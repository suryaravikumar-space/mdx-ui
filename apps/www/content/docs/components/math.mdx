## Installation

```bash
npx @ravikumarsurya/mdx-ui add math
```

Add KaTeX CSS to your root layout:

```tsx
import "katex/dist/katex.min.css"
```

## Two authoring paths

The math system supports two approaches that can be mixed freely:

| Approach | Best for |
|----------|----------|
| `<M>` / `<BM>` | Writing LaTeX strings — human authors who know LaTeX, AI-generated content |
| [Math Primitives](/docs/components/math-primitives) | Structured math UI — numbered equations, solution steps, custom layout |

## M and BM (recommended)

`M` is inline math. `BM` is block (display) math. Both accept LaTeX via the `expr` prop — backslashes and braces are **literal**, no JS string escaping needed.

```mdx
<M expr="\frac{a}{b}" />

<BM expr="x = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}" />
```

### Inline

The quadratic formula gives <M expr="x = \frac{-b \pm \sqrt{b^2-4ac}}{2a}" /> for <M expr="ax^2 + bx + c = 0" />.

### Block

<BM expr="e^{i\pi} + 1 = 0" />

<BM expr="\int_{-\infty}^{\infty} e^{-x^2}\,\d x = \sqrt{\pi}" />

<BM expr="\nabla \times \vec{E} = -\frac{\partial \vec{B}}{\partial t}" />

## InlineMath / BlockMath

The longform components are still available and accept a JS string as `children`. Useful when you need to embed a dynamic value.

```mdx
<InlineMath>{"E = mc^2"}</InlineMath>

<BlockMath>{"F = G\\frac{m_1 m_2}{r^2}"}</BlockMath>
```

Note the double backslashes — required because the value is a JS string literal inside `{"..."}`. Use `M`/`BM` with `expr` to avoid this.

## Built-in macros

These shorthands are available in any math expression:

| Macro | Expands to | Example |
|-------|-----------|---------|
| `\R`, `\N`, `\Z`, `\C` | `\mathbb{R/N/Z/C}` | `x \in \R` |
| `\Q`, `\F` | `\mathbb{Q}`, `\mathbb{F}` | `q \in \Q` |
| `\E` | `\mathbb{E}` | `\E[X]` |
| `\PP` | `\mathbb{P}` | `\PP(A)` |
| `\d` | `\,\mathrm{d}` | `\d x` |
| `\eps` | `\varepsilon` | `\eps > 0` |
| `\norm{v}` | `\left\|v\right\|` | `\norm{x - y}` |
| `\abs{x}` | `\left|x\right|` | `\abs{f(x)}` |
| `\inner{u}{v}` | `\langle u, v \rangle` | `\inner{x}{y}` |
| `\set{x \mid P}` | `\{x \mid P\}` | `\set{n \mid n > 0}` |
| `\floor{x}` | `\lfloor x \rfloor` | `\floor{n/2}` |
| `\ceil{x}` | `\lceil x \rceil` | `\ceil{\log n}` |

### Examples using macros

<BM expr="\forall x \in \R,\; \norm{x} \geq 0" />

<BM expr="\E[X] = \int_{-\infty}^{\infty} x\, f(x)\,\d x" />

<BM expr="\inner{u}{v} = \sum_{i=1}^{n} u_i v_i" />

## Examples

### Calculus

<BM expr="\frac{\d}{\d x}\left[\int_a^x f(t)\,\d t\right] = f(x)" />

<BM expr="\sum_{n=0}^{\infty} \frac{x^n}{n!} = e^x" />

### Linear algebra

<BM expr="\det(A) = \sum_{\sigma \in S_n} \text{sgn}(\sigma) \prod_{i=1}^n a_{i,\sigma(i)}" />

### Statistics

<BM expr="\PP(A \mid B) = \frac{\PP(B \mid A)\,\PP(A)}{\PP(B)}" />

<BM expr="\text{Var}(X) = \E[X^2] - \E[X]^2" />

### Number sets

Sets like <M expr="\R" />, <M expr="\N" />, <M expr="\Z" />, <M expr="\Q" />, <M expr="\C" /> use the built-in macros — no `\mathbb{}` needed.

## Props

### M

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `expr` | `string` | — | LaTeX expression (backslashes literal) |
| `className` | `string` | — | Additional CSS classes |

### BM

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `expr` | `string` | — | LaTeX expression (backslashes literal) |
| `className` | `string` | — | Additional CSS classes |

### Math / InlineMath / BlockMath

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `string` | — | LaTeX expression (JS string — escape backslashes) |
| `block` | `boolean` | `false` | Display mode (Math only) |
| `className` | `string` | — | Additional CSS classes |
